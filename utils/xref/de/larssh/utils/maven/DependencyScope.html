5    * Dependency scope is used to limit the transitivity of a dependency, and also
6    * to affect the class path used for various build tasks.
7    *
8    * <p>
9    * As described at <a href=

12  public enum DependencyScope {
13  	/**
14  	 * This is the default scope, used if none is specified. Compile dependencies
15  	 * are available in all class paths of a project. Furthermore, those
16  	 * dependencies are propagated to dependent projects.
17  	 */
18  	COMPILE("compile"), /**
19  	 * This is much like {@link #COMPILE}, but indicates you expect the JDK or a
20  	 * container to provide the dependency at runtime. For example, when building a
21  	 * web application for the Java Enterprise Edition, you would set the dependency
22  	 * on the Servlet API and related Java EE APIs to scope {@code provided} because
23  	 * the web container provides those classes. This scope is only available on the
24  	 * compilation and test class path, and is not transitive.
25  	 */
26  	PROVIDED("provided"), /**
27  	 * This scope indicates that the dependency is not required for compilation, but
28  	 * is for execution. It is in the runtime and test class paths, but not the
29  	 * compile class path.
30  	 */
31  	RUNTIME("runtime"), /**
32  	 * This scope indicates that the dependency is not required for normal use of
33  	 * the application, and is only available for the test compilation and execution
34  	 * phases. This scope is not transitive.
35  	 */
36  	TEST("test"), /**
37  	 * This scope is similar to {@link #PROVIDED} except that you have to provide
38  	 * the JAR which contains it explicitly. The artifact is always available and is
39  	 * not looked up in a repository.
40  	 */
41  	SYSTEM("system"), /**
42  	 * This scope is only supported on a dependency of type {@code pom} in the
43  	 * {@code dependencyManagement} section. It indicates the dependency to be
44  	 * replaced with the effective list of dependencies in the specified POM's
45  	 * {@code dependencyManagement} section. Since they are replaced, dependencies
46  	 * with a scope of {@code import} do not actually participate in limiting the
47  	 * transitivity of a dependency.
48  	 */
